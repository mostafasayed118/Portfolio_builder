## What I do

- Provide ready-to-use DESIGN.md files extracted from real websites
- Each DESIGN.md captures a complete visual system in plain text
- Enable AI agents to generate consistent, pixel-perfect UIs
- Cover 9 sections: visual theme, colors, typography, components, layout, depth, do's/don'ts, responsive behavior, and agent prompts

## What's inside each DESIGN.md

Every file follows the Stitch DESIGN.md format with extended sections:

1. **Visual Theme & Atmosphere** - Mood, density, design philosophy
2. **Color Palette & Roles** - Semantic name + hex + functional role
3. **Typography Rules** - Font families, full hierarchy table
4. **Component Stylings** - Buttons, cards, inputs, navigation with states
5. **Layout Principles** - Spacing scale, grid, whitespace philosophy
6. **Depth & Elevation** - Shadow system, surface hierarchy
7. **Do's and Don'ts** - Design guardrails and anti-patterns
8. **Responsive Behavior** - Breakpoints, touch targets, collapsing strategy
9. **Agent Prompt Guide** - Quick color reference, ready-to-use prompts

## How to use

1. Browse `skills/awesome-design-md/design-systems/` for available designs
2. Copy a site's `DESIGN.md` into your project root
3. Tell your AI agent to use it when building UI components

## Available design systems

Browse the `awesome-design-md/design-systems/` directory for designs from:
- Developer tools (Cloudflare, VoltAgent, etc.)
- Design & creative tools (Figma, Framer, Webflow, etc.)
- And many more brand design systems

Each design includes:
- `DESIGN.md` - The design system document
- `preview.html` - Visual catalog (light mode)
- `preview-dark.html` - Visual catalog (dark mode)
