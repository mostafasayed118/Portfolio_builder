==================================================
WHAT TO REVIEW
==================================================

Audit the entire codebase across these categories:

### 1. Architecture & Structure
- Is the folder structure clean and scalable?
- Are concerns separated correctly?
- Is business logic mixed inside UI components?
- Are reusable utilities extracted properly?
- Are components too large or too coupled?
- Are there files in the wrong place?
- Are naming conventions clear and consistent?

### 2. Code Quality
- TypeScript quality
- any `any` abuse
- poor inferred types
- duplicated code
- dead code
- unreachable code
- overcomplicated logic
- poor state handling
- inconsistent patterns
- missing loading / empty / error states
- risky assumptions in code

### 3. React / Next.js Best Practices
- server vs client component boundaries
- unnecessary `"use client"`
- improper hooks usage
- excessive client-side rendering
- poor data fetching patterns
- bad route organization
- unnecessary rerenders
- bad prop drilling
- missing memoization where justified
- incorrect use of dynamic imports
- improper image handling
- improper font loading
- metadata / SEO best practices
- caching and rendering strategy issues

### 4. Performance Review
Audit for performance issues including:
- unnecessary rerenders
- large bundle risks
- heavy client components
- large dependencies imported globally
- poor code splitting
- poor lazy loading
- blocking scripts
- unoptimized images
- missing dimensions causing layout shift
- expensive computations in render
- unnecessary effects
- event listener cleanup issues
- memory leaks
- poor table/list rendering
- animation performance issues
- expensive libraries where lighter alternatives exist

### 5. Core Web Vitals Mindset
Review likely issues affecting:
- LCP
- CLS
- INP / responsiveness
- TTFB where relevant

Check for patterns known to hurt Core Web Vitals, such as:
- large above-the-fold payloads
- unoptimized media
- unstable layout
- excessive JS on first load
- poor loading strategy for third-party scripts

### 6. Accessibility
Check for:
- semantic HTML issues
- missing labels
- poor keyboard support
- weak focus states
- contrast concerns if visible in code
- bad aria usage
- inaccessible dialogs/forms/tables
- reduced motion concerns
- screen-reader issues

### 7. Security / Reliability Basics
Check for:
- unsafe rendering
- user input assumptions
- missing validation
- weak form handling
- unsafe external links
- weak error handling
- missing fallback states
- bad environment variable handling
- sensitive data exposure risks

### 8. Styling / UI System Quality
Check whether:
- design tokens are consistent
- styling is scalable
- class usage is maintainable
- there is unnecessary duplication in styles
- dark mode is implemented safely
- theme logic is robust
- component variants are consistent
